# 킥오프: The Truman Show 앱 (가칭)

## 컨셉

"나도 몰랐던 내 삶을 누군가 방송 중이다."
사용자의 하루 데이터(사진, 위치, 스크린타임)를 자동 수집해, 매일 밤 3인칭 방송 내레이션 형식의 "에피소드"(S01E01...)를 생성하는 iOS 앱.
핵심 정체성: **입력 UI가 없는 것.** 사용자는 밤에 푸시 알림을 받고 에피소드를 '시청'만 한다. 일기 앱이 아니라 관찰 예능이다.

## 톤 & 세계관 규칙 (내레이션 프롬프트 설계 시 준수)

- 3인칭 방송 내레이션. 무미건조한 데이터를 드라마틱하게 과장한다. ("인스타그램 2시간" → "주인공의 도피는 오후 내내 계속됐다")
- 사진첩 사진은 주인공이 '찍은' 것 → 주인공은 기본적으로 카메라 뒤에 있다. 사진 속 인물은 전부 "출연진/조연/특별 게스트"로 서술.
- 데이터가 없는 날도 콘텐츠다: "주인공이 오늘 카메라에 잡히지 않았다. 제작진 사이에 정적이 흘렀다. 시청률 급락." (결측 = 방송 사고 에피소드)
- 에피소드 구성: 내레이션(장면 단위) + 시청률 + 하이라이트(그날 사진) + 가짜 시청자 댓글 3개 + 하단 "제작 리포트"(스크린타임, 온디바이스 전용).
- **앱 전체 UX 라이팅도 "제작진/PD 톤"으로 통일한다.** 애정 어린 관찰 예능 톤이지 공포·감시 톤이 아니다 ("난 너를 지켜보고 있다" 방향 금지 — 권한 허용률과 심사에 모두 불리). 권한 요청, 설정, 에러 메시지까지 제작진이 주인공에게 보내는 메모 형식.
  - 권유 멘트 예시 (사진 권한): "제작진 요청 📋 — 요즘 에피소드가 밋밋하다는 시청자 의견이 있습니다. 카메라(사진첩)를 열어주시면 훨씬 생생한 장면을 담을 수 있어요. 사진 원본은 기기 밖으로 나가지 않고, 제작진은 분석된 텍스트만 봅니다. — 담당 PD 드림"
  - 단, 이런 카피는 시스템 권한 팝업 **전에 띄우는 커스텀 프리퍼미션 화면**에만 사용. iOS purpose string(Info.plist)은 애플 심사 기준에 맞게 건조하고 사실적으로 작성.

## 아키텍처 결정사항 (확정)

- **네이티브 SwiftUI** (Expo/RN 아님 — Screen Time API와 Extension 때문에 필수)
- **데이터 수집 (반자동, 입력 최소화)**
  - 사진: PhotoKit으로 그날 사진 자동 수집 + EXIF 시간·GPS 추출
  - 위치: CLVisit (백그라운드 방문 감지, 저전력)
  - 스크린타임: DeviceActivityReport Extension (아래 제약 참고)
- **LLM: Foundation Models 프레임워크 (WWDC26 버전)**
  - 온디바이스 모델이 이미지 입력 지원 + 한 API로 온디바이스/Private Cloud Compute/Claude 등 클라우드 스위칭 가능
  - 역할 분담: 온디바이스 = 사진 캡셔닝·태깅·얼굴 매칭 / 클라우드 = 창작 내레이션 (한국어 창작 품질 때문)
  - 프라이버시 모드(전부 온디바이스) vs 화질 모드(클라우드 내레이션) 토글 제공
- **구형 기기 폴백 (iPhone 14 이전, Apple Intelligence 미지원)**
  - FM 캡셔닝 대신 **Vision 프레임워크**(전 기기 지원)로 온디바이스 장면 분류·객체 태깅·얼굴 감지 → 거친 태그("음식, 실내, 인물 2명")라도 "사진은 기기 밖으로 안 나감" 원칙을 전 기기에서 유지
  - 옵션으로 **"생생 모드" 옵트인**: 명시 동의 시 축소된 사진을 클라우드 API로 전송해 풍부한 캡셔닝. 기본값은 항상 오프. 디바이스 capability 감지 → 캡셔닝 엔진(FM/Vision/생생 모드)을 프로토콜로 추상화해 교체 가능하게 설계
- **저장: 로컬 전용** (SwiftData 또는 SQLite). 계정 없음, 서버 DB 없음.
- 클라우드 API 쓸 경우: 앱에 키 임베드 금지 → 경량 프록시(Cloudflare Workers 등) 경유, 무로깅

## 프라이버시 설계 (핵심 차별점)

- 사진 원본은 기기 밖으로 안 나감. 온디바이스에서 텍스트 캡션·태그로 변환 후 **텍스트만** 클라우드 전송
- GPS 좌표 대신 온디바이스 역지오코딩한 동네 이름만 전송
- 투명성 화면 "제작진이 본 것": 이번 에피소드 생성에 실제 전송된 텍스트를 그대로 공개 (감시 테마와 컨셉적으로도 맞음)

## 스크린타임 제약 (중요)

- DeviceActivityReport Extension은 샌드박스: 네트워크 차단, 본체 앱으로 데이터 반출도 공식적으론 불가
- 따라서 스크린타임은 **절대 LLM API로 보내지 않는다** (심사 리젝 사유)
- 처리: Extension 내부에서 룰셋 기반 한 줄 코멘트 생성 → 에피소드 하단 "제작 리포트" 섹션으로 렌더링
- Family Controls 개발용 entitlement는 즉시 발급, 배포용은 애플 승인 필요(거절 가능성 높음) → 스크린타임 없이도 성립하는 앱으로 설계, 이 기능은 optional 모듈로 격리

## 인물 인식

- 온보딩 셀카 등록은 선택사항. 스킵 시 "주인공은 카메라 뒤" 가정으로 진행, 오탐 허용
- 에피소드에서 얼굴 탭 → "이 사람 나야"로 lazy 등록, 이후 레퍼런스로 축적
- 오탐은 세계관으로 소화: "제작진이 주인공을 잘못 알아봤다"
- v2: Vision 프레임워크 온디바이스 얼굴 클러스터링 → 반복 등장인물 랭킹("시즌 개근 조연")

## 권한 거절 대응 (플로우 분기)

- **사진 거절 시**: 데이터 제로 방지를 위해 CLVisit 위치 수집을 이 경로에서는 MVP로 승격. 위치 (+ 가능하면 스크린타임)만으로 에피소드 생성.
- 부실한 에피소드도 세계관으로 소화: "카메라가 꺼진 세트장. 제작진은 위치 신호에만 의존해 방송을 이어가고 있다." — 결과물 자체가 사진 허용의 동기가 되도록.
- 에피소드 뷰 안에 제작진 톤의 재권유 배너 노출 (위 권유 멘트 예시 참조). 시스템 팝업 재요청은 설정 앱 유도로.

## MVP 범위 (주말 스코프)

1. 온보딩: 이름 입력, 권한 요청(사진, 위치), 셀카 등록(스킵 가능)
2. 그날 사진 자동 수집 → EXIF 파싱 → 온디바이스 캡셔닝 → 타임라인 구성
3. 에피소드 생성 파이프라인 (LLM 호출, 위 톤 규칙 반영한 프롬프트)
4. 에피소드 뷰 (TV 프로그램 스타일, SxxExx 넘버링, 썸네일 = 그날 사진)
5. 아카이브 리스트 + 밤 푸시 알림 (로컬 노티)

## MVP에서 제외 (v2+)

- 스크린타임 Extension (제약 검증 후 별도 작업)
- CLVisit 위치 타임라인 (사진 허용 유저는 EXIF 위치로 대체. 단, 사진 거절 경로에서는 MVP — 위 "권한 거절 대응" 참조)
- 대시보드(시청률 추이, 등장인물 통계), 얼굴 클러스터링, 공유 기능, Bedrock 실험
- 안드로이드: iOS 검증 전까지 보류. 간다면 Kotlin 네이티브 별도 앱 + 프롬프트/세계관 설계만 공유 (참고: UsageStatsManager로 스크린타임 직접 읽기 가능해 iOS보다 제약 적음)

## 첫 작업 지시

Xcode 프로젝트 스캐폴딩부터 시작하되, 다음 순서로:
1. SwiftUI 프로젝트 구조 + SwiftData 모델 (Episode, Scene, CastMember)
2. PhotoKit 수집 + EXIF 파싱 서비스
3. Foundation Models 연동 (온디바이스 캡셔닝 먼저, 폴백: 클라우드)
4. 에피소드 생성 프롬프트 템플릿 (위 세계관 규칙 반영, 별도 파일로 분리해 반복 튜닝 가능하게)
5. 에피소드 뷰 UI

각 단계마다 실기기 테스트 가능한 상태로 커밋. 시뮬레이터는 사진/스크린타임 데이터가 제한적이니 실기기 기준으로 안내할 것.
